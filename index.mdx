capa analyzes executable files and tells you what they can do. Point it at a PE, ELF, .NET module, shellcode file, or sandbox report, and it reports detected behaviors — mapped to MITRE ATT&CK and Malware Behavior Catalog (MBC) frameworks.

## What capa does

Run capa against an unknown binary and it reports exactly what capabilities it detected — and where:

```
$ capa suspicious.exe

+--------------------+------------------------------------------------------------------------+
| ATT&CK Tactic      | ATT&CK Technique                                                       |
|--------------------+------------------------------------------------------------------------|
| DEFENSE EVASION    | Obfuscated Files or Information [T1027]                                |
| DISCOVERY          | Query Registry [T1012]                                                 |
|                    | System Information Discovery [T1082]                                   |
| EXECUTION          | Command and Scripting Interpreter::Windows Command Shell [T1059.003]   |
| PERSISTENCE        | Create or Modify System Process::Windows Service [T1543.003]           |
+--------------------+------------------------------------------------------------------------+

+-------------------------------------------+-------------------------------------------------+
| CAPABILITY                                | NAMESPACE                                       |
|-------------------------------------------+-------------------------------------------------|
| execute shell command and capture output  | c2/shell                                        |
| connect to HTTP server (3 matches)        | communication/http/client                       |
| encode data using XOR (6 matches)         | data-manipulation/encoding/xor                  |
| run as a service                          | executable/pe                                   |
| create service                            | host-interaction/service/create                 |
+-------------------------------------------+-------------------------------------------------+
```

## Analysis backends

capa supports multiple backends for feature extraction:

| Backend | Use case |
|---------|----------|
| Vivisect (default) | Static analysis of PE and ELF files |
| IDA Pro | Interactive analysis via the capa Explorer plugin |
| Ghidra | Open-source alternative to IDA using PyGhidra |
| Binary Ninja | Analysis using the Binary Ninja engine |
| BinExport2 | Cross-tool analysis via BinExport2 format |
| CAPE / DRAKVUF / VMRay | Dynamic analysis from sandbox reports |
| .NET (dnfile) | Analysis of .NET managed code |
